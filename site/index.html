Wardex

The private-cloud XDR platform that puts detection, investigation, fleet operations, threat hunting, and governance under your control.

Wardex delivers a professional-grade security operations platform without cloud dependencies or vendor lock-in. 93 Rust modules cover cross-platform telemetry, Sigma and kernel-level detection, campaign clustering, memory forensics, deception engines, UEBA with geo-validation, file integrity monitoring, digital twin calibration, and privacy-preserving federated learning — all in a single deployable binary with a live browser console.

Extraction

Advanced Analytics

Intelligence-Driven Security

Multi-signal fusion, behavioral baselines, and privacy-preserving machine learning bring advanced analytics to your private cloud — without sending data to third parties.

01

Side-Channel Score Fusion

Combine traditional detection signals with hardware side-channel analysis. Critical findings boost compound threat scores by up to +1.5 for higher-confidence alerting.

02

EWMA Drift Tracking

Exponentially weighted moving average adaptation of device fingerprints catches gradual behavioral drift that static baselines miss entirely.

03

Impossible Travel Detection

Geo-IP validation flags authentication events that imply travel speeds exceeding 900 km/h, catching credential sharing and account compromise.

04

Digital Twin Calibration

Snap virtual device replicas to real-world telemetry snapshots and quantify per-parameter drift for continuous accuracy validation.

05

Sigma–Kernel Bridge

Map raw kernel events — process creation, file writes, DNS queries, network connections — directly into Sigma field schemas for unified rule evaluation.

06

Federated Convergence

Train global detection models across distributed agents without centralizing sensitive data. Multi-round averaging converges to target accuracy thresholds automatically.

Deployment

Release and Operate

Wardex is designed for self-hosted deployment. Build locally for development, tag and ship packaged releases for production, and use the browser console for day-to-day operations.

Build the platform

cargo build --release

Build the Rust binary locally for development and validation before packaging or tagging a release.

Launch the control plane

cargo run -- serve

Run the live HTTP server and browser console locally on port 8080.

Package releases

git tag v0.39.1

Tagged releases trigger GitHub Actions packaging for Linux, macOS, and Windows archives.

Validate the product

cargo test

Run the 991-test suite, then use the live checks and browser smoke for a release-quality verification pass.

Run as a service

deploy/wardex.service

Service packaging is included for systemd and launchd, with release archives bundling the site and examples.

Documented operations

docs/runbooks/

Operators get runbooks for response, SIEM integration, and platform-specific agent handling.
