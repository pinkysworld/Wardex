Wardex

Enterprise detection, investigation, fleet operations, and governance in one deployable control plane.

Wardex is built for teams that want a professional XDR/SIEM experience without outsourcing their control plane. It combines cross-platform telemetry, managed detection content, analyst workbench workflows, approval-gated response, agent release operations, and enterprise supportability in a single Rust product with a live browser console.

Architecture

Platform Flow

Wardex is organized around operator flow rather than isolated features: collect and score telemetry, work incidents, govern response, and ship managed releases from the same control plane.

Cross-platform agents and the local server collect telemetry, process context, integrity signals, and fleet health from macOS, Linux, and Windows environments.

Deployment

Release and Operate

Wardex is designed for self-hosted deployment. Build locally for development, tag and ship packaged releases for production, and use the browser console for day-to-day operations.

Build the platform

cargo build --release

Build the Rust binary locally for development and validation before packaging or tagging a release.

Launch the control plane

cargo run -- serve

Run the live HTTP server and browser console locally on port 8080.

Package releases

git tag v0.31.0

Tagged releases trigger GitHub Actions packaging for Linux, macOS, and Windows archives.

Validate the product

cargo test

Run the 692-test suite, then use the live checks and browser smoke for a release-quality verification pass.

Run as a service

deploy/wardex.service

Service packaging is included for systemd and launchd, with release archives bundling the site and examples.

Documented operations

docs/runbooks/

Operators get runbooks for response, SIEM integration, and platform-specific agent handling.
