Before You Email

Make the first reply useful.

01

Version and install channel

Include the Wardex version plus whether you installed from source, release archive, APT, RPM, Homebrew, container image, or Helm.

02

Deployment shape

Say whether this is evaluation, a live control plane, an air-gapped rollout, or a staged fleet deployment.

03

Exact failure and expected result

Attach the failing route, page, or command plus what you expected to happen instead.

04

Proof artifacts

Logs, screenshots, request IDs, support bundles, and deployment trust reports save a lot of back-and-forth.
